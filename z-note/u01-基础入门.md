# 1. 概念

**概念：** HTML5(Hyper Text Mark-up Language 5)：超文本标记语言5：
- 超文本：文字、图片、音频、视频、链接...
- 标记：浏览器根据标记分析功能从而完成显示。
- 语言：HTML是一门面向浏览器的语言。

# 1.1 H5发展历史

**概念：**
- 启航：HTML1 - HTML4
    - HTML第一个官方版本是由IETF（互联网工程任务组）推出的，后来W3C取代IETF继续研究HTML语言，频繁地推陈出新。
        - 四流公司：做项目的。
        - 三流公司：做产品的，用友，华为，小米。
        - 二流公司：做服务的，联通，移动，电信。
        - 一流公司：做标准的，W3C，JS标准公司。
    - HTML 3.20：1997年01月14日，W3C推荐标准。  
    - HTML 4.00：1997年12月18日，W3C推荐标准。
    - HTML 4.01：1999年12月24日，W3C推荐标准，版本相对稳定。
- 决裂：XHTML1 - XHTML2
    - XHTML1.0是继HTML4.01后的第一个修订版本，是具有XML风格的HTML，相对HTML并没有引入新标签或者属性，唯一改变就是语法更加严格。
    - 之后W3C又推出了XHTML1.1和2.0版本，语法愈加严格完美，但随之而来的问题也愈加严重，因为W3C改动太大而且HTML不向前兼容，导致各大浏览器厂商跟不上W3C的脚步。
    - 最终，浏览器厂商自己成立了超文本应用技术工作组：WHATWG，正式与W3C决裂。
- 和好：WebApps1.0
    - 自从双方决裂，WHATWG致力于研究WebForms2.0和WebApps1.0（后合并为HTML5），而W3C继续研究XHTML2，但困难却越来越大，双方的日子都不好过。
    - 06年，W3C成立了一个新的HTML工作组，并明智地选择了WHATWG的成果作为基础，他们同时进行着XHTML2和HTML5两套规范。
    - 09年，W3C终于坚持不住，主动宣布：XHTML时代已死，XHTML语法永存。于是，W3C和WHATWG言归于好，开始共同研究HTML5，也就是HTML语言目前最新的版本。
- 爆发：HTML5
    - HTML5草案的前身名为 Web Applications 1.0，于2004年被WHATWG提出，于2007年被W3C接纳，并成立了新的HTML工作团队。
    - HTML5 的第一份正式草案已于2008年1月22日公布。HTML5仍处于完善之中，然而，大部分现代浏览器已经具备了某些HTML5支持。
    - 2012年12月17日，万维网联盟（W3C）正式宣布凝结了大量网络工作者心血的HTML5规范已经正式定稿，根据W3C的发言稿称，HTML5是开放的Web网络平台的奠基石。
    - 2013年5月6日， HTML5.1正式草案公布，该规范定义了第五次重大版本，第一次要修订万维网的核心语言：超文本标记语言（HTML），在这个版本中，新功能不断推出，以帮助Web应用程序的作者，努力提高新元素的操作性。
    - 2014年10月29日，万维网联盟泪流满面地宣布，经过几乎8年的艰辛努力，HTML5标准规范终于最终制定完成了，并已公开发布。

# 1.2 H5浏览器支持

**概念：**
- 支持H5的浏览器包括火狐浏览器，IE9及其更高版本，谷歌浏览器，Safari，Opera等。
- 国内的傲游浏览器（Maxthon），以及基于IE或Chromium（Chrome的工程版或称实验版）所推出的360浏览器、搜狗浏览器、QQ浏览器、猎豹浏览器等国产浏览器同样具备支持HTML5的能力。 

# 1.3 H5的移动端开发

**概念：**
- 在移动设备开发HTML5应用只有两种方法，要不就是全使用HTML5的语法，要不就是仅使用JavaScript引擎。
- JavaScript引擎的构建方法让制作手机网页游戏成为可能，由于界面层很复杂，已预订了一个UI工具包去使用。
- 纯HTML5手机应用运行缓慢并错漏百出，但优化后的效果会好转，尽管不是很多人愿意去做这样的优化，但依然可以去尝试。
- HTML5手机应用的最大优势就是可以在网页上直接调试和修改，原先应用的开发人员可能需要花费非常大的力气才能达到HTML5的效果，不断地重复编码、调试和运行，这是首先得解决的一个问题，因此也有许多手机杂志客户端是基于HTML5标准，开发人员可以轻松调试修改。

# 1. 文件

**概念：** 
- H5文件都是以.html或.htm结尾的，二者的主要区别是优先级，.html优先级更高，使用时，推荐使用.html作为HTML文件的后缀。
- 创建HTML文件的方法非常简单，新建文本文档，命名为：index.html

# 2. 标签

**概念：** 
- HTML文档由HTML标签和纯文本组成，标签描述功能，文本描述内容，而它的强大功能主要就体现在标签对文本的控制方面。	
- 标签也叫元素、标记等。

## 2.1 组成

**概念：** 标签是由"<>"和小写英文字母组成的，如 `<body>` 、 `<title>` 等。

## 2.2 分类

**概念：** 
- 标签的分类有三种：单标签、双标签和组合标签
- 单标签一般不配合文本使用，如案例中的 `<br />` 代表换一行。
- 单标签要以 `/>` 结尾，建议标签名和结尾的 `/>` 之间空一格。
- 双标签成对儿出现，用来描述文本或定义文本。

分类     | 格式                  | 举例
-        |-                      |-
单标签   | <标签名 />            | `<br />`
双标签   | <标签名>文本</标签名> | `<p>文本</p>`
组合标签 | <父标签><br /><子标签>文本</子标签><br /></父标签> | `<select><option>文本</option></select>`

## 2.3 属性

**概念：** 

- 标签中通过键值对儿（名值对儿）来描述属性，如SIZE="7"、color="red"等。
- HTML不区分大小写，但XHTML要求属性名使用小写字母。

```
格式1：属性名 = "属性值"
格式2：属性名 = '属性值'
格式3：属性名 = 属性值，并不建议。
```

# 3. 回车

**概念：** 在编辑器中，代码上的回车效果是无效的，我们需要使用 `<br />` 来达到换行的效果。

# 4. 分割线

**概念：** 
- `<hr />` 分割线就是一条水平线，默认宽度占父元素的100%。
- `<hr />` 可以设置size、width和color来分别控制高度，宽度和颜色，但实际中，这些样式都用CSS代替。

**布局：**
```html
<hr color="red" width="50px"/>
```

# 1. 文档结构

**概念：** 
- 如下是一个HTML文件内部的基础结构标签。
- lang="zh-CN/en"告诉浏览器本页面是简体中文/英文页面，主要是方便某些浏览器的翻译功能。
- charset="utf-8"告诉浏览器本页面以utf-8方式编码，此时浏览器会默认使用utf-8的模式去显示你的页面，以免产生乱码。另外utf-8兼容性好于utf8写法。
- html4版本中的字符设置为：`<meta http-equiv="charset" content="text/html charset=utf-8">`

**布局：**
```html
<!DOCTYPE html>                                    <!--文档声明类型-->
<html lang="zh-CN">                                <!--HTML文档开始标签-->
    <head>                                         <!--页面头开始标签-->
        <meta charset="utf-8">                     <!--声明字符编码-->
        <title>documentType</title>                <!--设置文档标题-->
    </head>                                        <!--页面头结束标签-->
    <body>                                         <!--页面体开始标签-->
    ...                                            <!--HTML页面体内容-->
    </body>                                        <!--HTML页面体结束-->
</html>                                            <!--HTML文档结束标签-->
```

# 2. 文档声明

**概念：** 
- 文档声明[Document Type Declaration]负责声明HTML文档类型，告诉浏览器这是一个HTML文档，一般写在文档的第一行。
- 在H5之前的版本中我们有3种文档声明类型，表示具体的版本和风格，了解一下即可，而H5不再需要表示版本和风格，我们以后也只H5的文档声明，简单明了。

1. 严格型：这种文档对标记语法要求很严格。
```html
<!DOCTYPE html PUBLIC 
    "-//W3C//DTD HTML4.01//EN"
    "http://www.w3.org/TR/html4/strict.dtd>
```

2. 过渡型：这种文档对标记语法要求不是很严格，H5以前版本常用型。
```html
<!DOCTYPE html PUBLIC 
    "-//W3C//DTD HTML4.01 Transitiona//EN"
    "http://www.w3.org/TR/html4/loose.dtd>
```

3. 框架型：当网页设计中有框架元素，就用这种用文档类型。
```html
<!DOCTYPE html PUBLIC 
    "-//W3C//DTD HTML4.01 Frameset//EN"
    "http://www.w3.org/TR/html4/frameset.dtd>
```

4. 而我们的新文档声明：简洁明了
```html
<!DOCTYPE html>
```

# 1. 页头

**概念：** 
- head标签用于定义网页文档的头部，它是所有头部元素的容器。
- 在head标签中的元素可以使用 `<head>` 中：`<base>` 、 `<link>` 、 `<meta>` 、 `<script>` 、 `<style>` 以及 `<title>`。
- 绝大多数文档头部包含的数据都不会真正作为内容显示给读者，页面头只负责设置文档标题和其它在网页中不显示的信息。

**布局：**
```html
<head>
    <title>文档结构</title>
    <meta name="generator" content="sublime" />
    <base href="../image/" target="_blank"/>
    <style type="text/css"></style>
    <link rel="stylesheet" type="text/css" href="../style/my.css"/>
    <script></script>
</head>
```

# 2. 标题标签title

**概念：** 
- `<title>` 用来定义文档的标题，浏览器会以特殊的方式来使用标题，并且通常把它放置在浏览器窗口的标题栏或状态栏上。
- 当把文档加入用户的链接列表或者收藏夹或书签列表时，标题将成为该文档链接的默认名称。

# 3. 默认路径base

**概念：** 
- `base` 标签的作用就是给页面的链接加上默认的路径，或者默认的打开方式；
- 当设置了 `base` 的 `href` 值后，页面上所有的路径（ `href` 、 `src` 等都适用）都以该值为默认根目录（可覆盖）。
- 当设置了 `base` 的 `target` 值后，页面上所有的链接的打开方式都以该值为第一选择；
- `<base>`标签最好不要动态写入，因为在Firefox和IE中会不成功，但是Chrome加载正常

**布局：**
```html
<head>
    <base href="../image/" target="_blank"/>
    <title>HEAD标签测试</title>
</head>
<body>
    <img src="base_text.jpg" />
    <a href="http://www.baidu.com">百度</a>
</body>
```

> 上段代码在 `<head>` 中设置了 `href = "../image/"` ，那么 `<body>` 中 `<img>` 的路径就会自动补齐为 `../image/base_text.jpg` ，并且 `<a>` 的默认跳转方式都首选 `_blank`。

# 4. 指令信息meta

**概念：** 
- 用于提供有关页面的元信息，比如针对搜索引擎和更新频度的描述和关键词。
- 当服务器向浏览器发送文档时，会先发送许多名值对。其中至少包括一个：`content-type:text/html` ，告诉浏览器准备接收一个HTML文档，只有当浏览器可以接收这些附加的头部字段，并能以适当的方式使用它们时，这些字段才有意义。

**布局：**
```html
<!--网页关键字-->
<meta name="keywords" content="HTML,JAVA" />

<!--网页作者-->
<meta name="author" content="JoeZhou" />

<!--网页内容描述-->
<meta name="description" content="这是个门户网站" />

<!--网页生成工具-->
<meta name="generator" content="idea" />

<!--设置两秒跳入百度-->
<meta http-equiv="refresh" content="2;URL=http://www.baidu.com" />
```

> name不是必须有的，而是我们对自己和源文档的读者来说拟定的名称。如果没有name属性，名称值对中的名会采用http-equiv属性的值。

# 5. 外链link

**概念：** 用于链接外部css文件、链接收藏夹图标等，link标签最常见的用途是链接外部样式表，外部资源。

## 5.1 链接外部css文件

**布局：**
```html
<link rel="stylesheet" type="text/css" href="../style/link_text.css" />
```

## 5.2 链接页面标题图标

**布局：**
```html
<link rel="icon" type="image/x-icon" href="favicon.ico" /> 
```

> 图标文件名必须是favicon.ico，且最好放到网站的根目录下。

# 6.  脚本script

**概念：** 
- script脚本可以在 `<head>` 或 `<body>` 中编写执行。
- script写在 `<head>` 中，function只加载而不执行，需等某一事件触发才能执行。
- script写在 `<body>` 中，直接加载并执行。

# 7.  样式style

**概念：** 
- 为HTML文档定义样式信息，规定在浏览器中如何呈现HTML文档。
- 如需链接外部样式表，请使用link标签。

**布局：** 
```html
<style type="text/css">...</style>
```
