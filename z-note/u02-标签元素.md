# 1. 文本元素

**概念：** H5规定，所有的文本标签都要从语义出发，所以对H4的文本标签进行了升级。
- `<strong>`：文本加粗，用于高亮单词或短语，替换 `<b>`。
- `<em>`：文本倾斜，用于发言、技术词汇等，替换 `<i>`。
- `<del>`：删除线，用于标识过时和废弃的内容，替换 `<s>`。
- `<ins>`：下划线，用于强调和标记，替换 `<u>`。
- `<mark>`：标记突出显示文本，黄色背景黑色字。
- `<span>`：尽量不要让文本和其他标签同级，实在不知道添加什么标签，就用 `<span>`。
- `<small>`：将原本的字体缩小一号
- `<sup>`：将文字变为上标。
- `<sub>`：将文字变为下标。
- `<code>`：用于标识程序代码段。
- `<var>`：用于标识变量名。
- `<samp>`：用于标识输出内容。
- `<kbd>`：用于标识输入内容。
- `<q>`：用于其他地方的的内容。
- `<cite>`：用于其他地方的的标题。
- `<abbr>`：缩写，配合 `title` 属性展示全名。
- `<dfn>`：专业术语。
- `<bdo>`：设置文本方向：值有 `rtl` 和 `ltr`两个，默认 `ltr`。
- `<time>`：表示时间类型。

> 有些预设标签没有明显效果，是因为它英文范畴，需要改成 `lang="en"` 后，再进行测试。

**布局：** html/文本标签.html

# 2. 标题

**概念：** 
- 文本标题是对文字的一些预定义设置，从 `<h1>` 到 `<h6>` 分别有不同的大小设置，都有加粗、换行和行高的效果。
- `<font>` 在h5中被废除了。

**布局：**
```html
<h1>1号标题文本</h1>
<h2>2号标题文本</h2>
<h3>3号标题文本</h3>
<h4>4号标题文本</h4>
<h5>5号标题文本</h5>
<h6>6号标题文本</h6>
<h7>我是7号标题（失效，恢复成普通的文字）</h7>
```

**效果：**
<section style="border:1px dashed red; padding: 10px;">
<h1>1号标题文本</h1>
<h2>2号标题文本</h2>
<h3>3号标题文本</h3>
<h4>4号标题文本</h4>
<h5>5号标题文本</h5>
<h6>6号标题文本</h6>
<h7>我是7号标题（失效，恢复成普通的文字）</h7>
</section>

# 3. 段落

**概念：** 
- 标签 `<p>` 表示文本是一个段落，段落和段落之间自动换行并且保留一定空隙。
- 段落之中，最好只放文字相关的内容，这样才符合段落的语义，尽量不要嵌套其他容器元素。
- 段落可以不使用 `</p>` 进行结尾，但是建议还是写上。

属性 | 属性值 |描述
-|-|-
`contenteditable` | `false`<br/>`true` | 不可修改（默认）<br/>点击文本即可进行内容修改
`hidden`          | `hidden` | 代表将文本隐藏
`lang`            | `en`     | 设置局部语言为英语
`title`           | 字符串   | 当鼠标悬停在这个元素上的时候给出的文字提示
`dir`             | `rtl`<br />`ltr` | 文字居左显示<br/>文字居右显示

> 这些属性是全局属性，也可以作用在其他所有的标签上。

**布局：**
```html
<p>我是第一自然段</p>
<p>我是第二自然段</p>
<p contenteditable="true">我是小说中的一段文字</p>
<p hidden="hidden">你好世界</p>
<p title="这是个小说">我是小说中的一段文字</p>
<p dir="ltr">我是小说中的一段文字</p>
<p dir="rtl">我是小说中的一段文字</p>
```

**效果：**
<section style="border:1px dashed red; padding: 10px;">
<p>我是第一自然段</p>
<p>我是第二自然段</p>
<p contenteditable="true">我是小说中的一段文字</p>
<p hidden="hidden">你好世界</p>
<p title="这是个小说">我是小说中的一段文字</p>
<p dir="ltr">我是小说中的一段文字</p>
<p dir="rtl">我是小说中的一段文字</p>
</section>

# 1. 相对路径和绝对路径

**概念：**
- 绝对路径从电脑盘符出发，工程中很少使用。
- 相对路径从自身出发，常用符号有两个：
    - `./` ：代表当前位置时，可以省略。
    - `../` ：代表上层位置时，可以重复多次使用。

# 2. 外链接

**概念：** 外链接指的是跳入外部URL或者工程内其他HTML文件。

## 2.1 href属性

**概念：** 
- `href` 是 `<a>` 标签的核心属性，用来设置跳转路径，值为外部URL或者工程内其他文件的相对路径或绝对路径。
- 即使还没有决定属性值，也要写全属性名，否则没有超级链接的效果。
- `href` 的属性值可以使用 `#` ，表示当前页。

```html
<!-- a标签必须配合href属性才能有点击效果 -->
<a href = "http://www.baidu.com">点我跳入百度</a>
<a href = "b.html">工程内其他文件</a>
```

## 2.2 target属性

**概念：** `target` 决定了目标内容要显示在哪个页面或框架显示。

target值 | 描述
-|-
`_blank`   | 代表在新窗口显示
`_top`     | 代表在顶层窗口和框架中显示
`_self`    | 代表在当前窗口显示（默认）
`_parent`  | 代表在父窗口(frameset)中显示
`自定义`   | 代表在自定义名字的窗口或框架中显示如果自定义名字不存在，则在新窗口中显示

**布局：**
```html
<a href = "http://www.baidu.com" target="_self">本页面跳入百度</a>
<br />
<a href = "http://www.baidu.com" target="_blank">新页面跳入百度</a>
```

**效果：**
<section style="border: 1px dashed red; padding:10px;">
<a href = "http://www.baidu.com" target="_self">本页面跳入百度</a>
<br />
<a href = "http://www.baidu.com" target="_blank">新页面跳入百度</a>
</section>

# 3. 内链接

**概念：** 
- 链接内部路径代表链接到当前自身文件的某一位置。此时，过程涉及到两部分内容：
    - 锚点：在某个标签中使用`id` 来标记锚点，如： `<span id="abc">我是锚点</span>` 。
    - 触发：在某个超级链接的 `href`属性中，使用 `#锚点` 的格式，跳回锚点位置，如：`<a href="#abc">跳回锚点</a>`。

**布局：**
```html
<span id="firstLine">小说第一行</span>
<!-- 使用100个hr，目的是把页面的滚动条逼出来 -->
<hr />
<a href="#firstLine">回到小说第一行</a>
```

> 使用 `id` 标记锚点时，任何元素都可以标记锚点，使用 `name` 替换 `id` 来标记锚点也可以，但此情况下，你只能用 `<a>` 来标记锚点。

# 4. 电邮链接

**概念：** `<a>` 标签支持使用 `href="mailto:邮箱地址"` 的格式来开启本地默认邮件客户端，然后发送邮件。

```html
<a href="mailto:yy06200210@163.com">给我发送电子邮件</a>
```

# 1. 分割容器

**概念：** 
- 标签 `<div>` 表示文本独占一行，`<div>` 之间自动换行但是没有空隙。
- `<div>` 中可以嵌套其他容器。

**布局：**
```html
<div>我独占一行</div>
<div>
    <div>我被嵌套</div>
</div>
```

**效果：**
<section style="border: 1px dashed red; padding:10px;">
<div>我独占一行</div>
<div>
    <div>我被嵌套</div>
</div>
</section>

# 2. 引用容器

**概念：** 标签 `<blockquote>` 表示文本来自别处，而且大段出现，前后两端都会有缩

**布局：**
```html
<blockquote>我是引用内容</blockquote>
```

**效果：**
<section style="border: 1px dashed red; padding:10px;">
<blockquote>我是引用内容</blockquote>
</section>

# 3. 保留容器

**概念：** 
- 标签 `<pre>` 代表文本在编辑器中是怎样排版，显示时就怎样排版，空格和回车都会被保留。
- `<pre>` 标签中尽量顶格写内容，因为前面的缩格，也会被保留。

**布局：**

```html
<pre>
00000     000     00000 
  0      0   0    0
  0      0   0    0000
  0      0   0    0
000       000     00000
</pre>
```

**效果：**
<section style="border: 1px dashed red; padding:10px;">
<pre>
00000     000     00000 
  0      0   0    0
  0      0   0    0000
  0      0   0    0
000       000     00000
</pre>
</section>

# 4. 布局容器

**概念：** H5中的文档元素的大部分标签是没有任何效果的，完全是为了配合语义使用，主要作用是划分各个不同的内容，让整个布局清晰明快，具有语义，进一步替代 `div` 传统布局。

属性 | 描述
-|-
`<header>` |首部：可存放LOGO、标题、导航等
`<nav>` |导航
`<section>` |重要概念、主题
`<article>` |独立的内容
`<aside>` |周边内容、少有牵涉的内容
`<footer>` |尾部：可存放版权声明和友情链接等
`<address>` |文档或article的联系信息
`<details>` |用户展开这个区域可以获得详细内容，兼容性不好
`<summary>` |用在details元素中，表示该元素内容的标题或说明，兼容性不好

**布局：**
```html
<header>
    <span>头部位置，存放Logo和导航</span>
    <nav>这里是导航的区域</nav>
</header>
<section>
    <span>主题部分</span>
    <article>主题中的独立内容一</article>
    <article>主题中的独立内容二</article>
</section>
<aside>侧边栏区域</aside>
<footer>版权信息和友情链接</footer>
